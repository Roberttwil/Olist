This dashboard is connected to the real, public Olist Brazilian E-Commerce Dataset on Kaggle.

Database Relationship Map (ERD)

customer_id vs customer_unique_id

customer_id: A temporary key generated for each order session. Use this to JOIN orders and customers.

customer_unique_id: The actual identifier of a physical customer. Use this to count unique buyers or track repeat purchases.

order_purchase_timestamp vs shipping_limit_date

order_purchase_timestamp: The actual date and time when the order occurred. Use this for all sales, freight, or order volume date-filtering.

shipping_limit_date: The deadline/limit for the seller to ship the product. Do NOT use this as order date filters.
